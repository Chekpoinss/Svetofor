# Svetofor
# Traffic Control Hackathon MVP

Решение для подзадачи по анализу видеопотока робота: детекция светофора, распознавание его сигнала и вывод команды `STOP` / `GO!` перед желтой стоп-линией.

## Описание задачи

На вход программе подается видео с робота.  
Необходимо:

- детектировать светофор на кадре;
- определить его сигнал (`RED` / `GREEN`);
- определить желтую стоп-линию на дороге;
- выводить:
  - `STOP` — если робот подъезжает к стоп-линии и горит красный;
  - `GO!` — если робот подъезжает к стоп-линии и горит зеленый.

Сообщение должно появляться **только в тот момент, когда робот должен остановиться / проехать**, то есть **перед стоп-линией**.

## Как работает решение

Обработка кадра делится на две части:

- **верхняя половина кадра** — поиск светофора;
- **нижняя половина кадра** — поиск желтой стоп-линии.

### Детекция светофора
Светофор определяется через HSV-маски:

- красный цвет — две маски в HSV;
- зеленый цвет — одна маска в HSV.

После этого выполняются:

- морфологическое открытие;
- морфологическое закрытие;
- поиск самого крупного контура;
- построение bounding box.

### Детекция стоп-линии
Желтая стоп-линия также ищется по HSV-маске, после чего:

- маска очищается морфологией;
- ищется самый крупный контур;
- строится bounding box;
- вычисляется нижняя точка линии.

### Логика принятия решения
Для уменьшения дерганий и случайных ошибок распознавания используется сглаживание по нескольким последним кадрам.

Решение принимается так:

- если робот **еще не близко** к стоп-линии — ничего не выводится;
- если робот **близко** к стоп-линии:
  - `RED` → `STOP`
  - `GREEN` → `GO!`
